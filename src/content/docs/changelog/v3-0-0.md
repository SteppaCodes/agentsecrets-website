# AgentSecrets v3.0.0

This release marks a major milestone for AgentSecrets, introducing a built-in Model Context Protocol (MCP) server, a unified Active Credential Firewall enforcing secure credential use at runtime via a transient proxy, zero-disk configuration storage via native OS Keychain integration, and local password write-protection for production environments.

---

## What's New in v3.0.0

### Built-in Native MCP Server
AgentSecrets now ships with a fully integrated Model Context Protocol (MCP) server built directly into the CLI. This allows LLM agents and AI clients (such as Cursor or Claude Desktop) to interact with AgentSecrets securely:
* **Rich Tool Suite**: Exposes tools for credential operations (api_call, list_keys, check_key, get_coverage), context management (get_status, get_environment, switch_environment, pull_secrets), audit logs (get_proxy_logs, get_blocked_requests), and allowlist configurations (check_domain, get_allowlist).
* **Auto-Configuration**: Running `agentsecrets mcp install` automatically configures Claude Desktop, Cursor, and OpenClaw to use the native MCP server, completely removing the need for manual JSON editing.

### Core Security Controller and Transient Proxy
AgentSecrets is no longer just about secure secrets access and storage. It is now about secure credential use. The proxy acts as the active credential firewall.
* **Unified Enforcement**: Both the CLI `call` command and the built-in MCP server's `api_call` tool route requests through the local proxy.
* **Transient Proxy Flow**: If the background proxy daemon is not running, the client launches a transient local proxy instance on a free port, routes the call, enforces the domain allowlist and response redaction, records the telemetry, logs the audit entry, and shuts it down. This guarantees 100% security parity across all CLI and agent entry points, ensuring all necessary checks are enforced for safe credential access and use.

### Zero-Disk Configuration Storage (keychain-auth Integration)
To eliminate the disk-based configuration attack surface, AgentSecrets now stores all user authentication tokens (JWT access and refresh tokens) and decrypted workspace keys inside the native OS Keychain (macOS Keychain, Windows Credential Manager, or GNOME Keyring/KWallet on Linux) managed securely via the keychain-auth daemon.
* **Empty Local Files**: `~/.agentsecrets/token.json` and `~/.agentsecrets/config.json` no longer store sensitive values on disk. All keys are stripped, leaving configuration files clean of plaintext secrets.
* **Automatic Compatibility and Migration**: Upgrading from older versions is completely seamless. When the CLI reads from legacy config files, it automatically migrates the credentials to the OS Keychain and purges them from the disk files.

### Production Write-Protection
Setting or updating secrets in the `production` environment, or setting them globally across all environments (`--all-envs`), now requires local password verification:
* **Interactive Verification**: The CLI prompts the developer to enter their password. The hash is verified locally using a PasswordHash derived from email and password (SHA-256), maintaining a zero-knowledge local verification loop.
* **Securing Against Attackers**: This stops processes, malicious scripts, or hijacked tools from unauthorized modifications to production variables.

### Forensic Snapshot Log (Full Governance Log v1.0)
To support strict SOC 2 Type II, ISO 27001, and HIPAA compliance for AI agents, AgentSecrets introduces an advanced forensic auditing engine that captures the complete security boundary state at the exact millisecond of request execution:
* **SQLite Storage System**: Logs are stored locally in a high-performance SQLite database under `~/.agentsecrets/audit.db` using the structured `forensic_audit_events` schema.
* **Cryptographic Hash Chaining**: Every log entry is cryptographically linked to the previous entry using SHA-256 (`chain_hash = sha256(prev_id + current_id + created_at)`), guaranteeing log immutability and non-repudiation.
* **Deep Governance Snapshots**: Each log entry stores detailed, immutable component blocks:
  - `Event`: Execution details, method, path, response status, and agent process validation.
  - `Snapshot`: State of the allowlist, secrets in scope, agent capabilities, and keychain daemon status.
  - `Enforcement`: Chronological pass/fail trace log of each security layer evaluated (capabilities, allowlist, secret policy).
  - `Resolution`: Injection styling, SSRF safety status, and response redaction/scrubbing triggers.
* **New Forensic CLI Tools**:
  - `agentsecrets log verify`: Recalculates hashes chronologically across the entire SQLite log database and verifies sequence integrity, instantly flagging any deleted, edited, or reordered entries.
  - `agentsecrets log replay <id>`: Renders a visual, step-by-step reconstruction of the active credential firewall and environment state at the exact millisecond the request was processed.

### Telemetry and Insights
New metrics are collected locally to monitor proxy performance and usage:
* **secrets_resolved**: Tracks the count of individual secrets resolved by the proxy.
* **total_proxy_duration_ms**: Tracks total milliseconds spent on forwarding requests, providing performance and speed insights.

## Upgrading and Migration

### 1. Update the CLI
:::step
Download and install the latest agentsecrets binary (v3.0.0):
```bash
brew upgrade agentsecrets
# Or via npm/pip
npm install -g @the-17/agentsecrets
pip install --upgrade agentsecrets-cli
```
:::

### 2. Clear Stale Sessions
:::step
On your local machine, run logout to clear any legacy credentials and log in again to establish the keychain baseline:
```bash
agentsecrets logout
agentsecrets login
```
This automatically establishes the secure keychain credentials baseline for v3.0.0.
:::
