# AgentSecrets v3.0.0

AgentSecrets v3.0.0 marks a major milestone, expanding the **zero-knowledge credential infrastructure** to now host and enforce complete **credential governance** for AI agents, rather than just secrets management. By moving beyond simple secrets retrieval to active runtime governance, this release introduces structural layers that prevent prompt injection leaks, trace agent operations, and lock down production credentials.

With the introduction of the **transient proxy**, AgentSecrets enforces active credential security regardless of whether you remember to run the daemon. This is the difference between a security tool and a **security guarantee**.

---

## What's New in v3.0.0

### Agent Identity & Capabilities
LLM agents now run under a formal identity framework rather than having unauthenticated, unrestricted access.
* **Agent Capabilities**: Limit what specific secrets an agent token can access by setting strict whitelists and blacklists.
* **Granular Controls**: Use `agentsecrets agent policy set <agent-id> --allow PUBLIC_SECRET --deny PRIVATE_SECRET` to restrict access right at the proxy boundary.
* **Identity Tokens**: Generate secure identity tokens (`agt_...`) bound to these policies via `agentsecrets agent token issue`.
* *Read more in [Agent Capabilities](../proxy/agent-capabilities).*

### Secret-Level Policies
Define fine-grained, secret-specific usage rules that restrict how and where credentials can be used.
* **Domain & Method Constraints**: Block secrets from being sent to unauthorized domains or limit them to specific HTTP methods (e.g., allow `GET` but require approval for `POST`).
* **Request Permission Flow**: Configure policies to require manual developer approval at runtime (via `agentsecrets proxy approve`) for critical actions.
* *Read more in [Secret Policies](../proxy/secret-policies).*

### Forensic Governance Logs & Replay
Support compliance standards (SOC 2, ISO 27001, HIPAA) with a tamper-proof auditing engine.
* **SQLite Storage**: Logs are saved locally in a high-performance SQLite database under `~/.agentsecrets/audit.db`.
* **Cryptographic Hash Chaining**: Every log entry is chained to the previous one via SHA-256 (`chain_hash = sha256(prev_id + current_id + created_at)`), making log files mathematically immutable.
* **Log Verification & Replay**: Use `agentsecrets log verify` to instantly flag deleted, edited, or reordered entries, and `agentsecrets log replay <id>` to visually reconstruct the exact environment state at the millisecond of execution.
* *Read more in [Audit Logs](../proxy/logs).*

### Built-in Native Model Context Protocol (MCP) Server
Allows LLM agents and modern AI clients to interact with AgentSecrets out-of-the-box.

We ship with **19 built-in tools** allowing LLMs to manage credentials, query logs, verify security states, and orchestrate execution:
* **Credential Operations**: `api_call` (make authenticated requests securely), `list_keys` (view key names), `check_key` (existence check), and `get_coverage` (drift check across environments).
* **Context Management**: `get_status` (state report), `get_environment` (active env), `switch_environment` (change active env), `pull_secrets` (sync from cloud), `diff_secrets` (compare local/cloud), and `diff_environments` (drift comparison source/target).
* **Audit & Observability**: `get_proxy_logs` (read local logs), `get_blocked_requests` (list violations), `get_redaction_events` (list body redacted calls), and `get_audit_summary` (statistics & aggregation).
* **Agent Identity**: `get_agent_identity` (identity status) and `list_agent_tokens` (list tokens in workspace).
* **Allowlist**: `check_domain` (check if domain allowed) and `get_allowlist` (list allowed domains).
* **Rotation**: `rotate_key` (trigger credential rotation).

Running `agentsecrets mcp install` automatically configures Cursor, Claude Desktop, and OpenClaw without manual configuration editing.

*Read more in [Cursor integration](../integrations/cursor) and [Claude Desktop integration](../integrations/claude-desktop).*

### Transient Proxy & Outbound Security
A paradigm shift in active credential protection.
* **100% Security Parity**: If the background proxy daemon is not running, the client dynamically spawns a transient proxy on a free port, executes the request securely, and tears it down. Security policies are always enforced, even if you forget to start the proxy.
* **Local Pre-Shared Session Token**: Blocks unauthorized local processes or browser extensions from calling the proxy. The CLI secures and rotates this token automatically via the OS Keychain using the `X-AS-Session-Token` header.
* **SSRF & DNS Rebinding Defenses**: Intercepts connections targeting private subnet ranges (RFC 1918) or loopback addresses. Resolves target IPs directly to prevent DNS Rebinding.
* *Read more in [SSRF Protection](../proxy/ssrf-protection) and [Session Token](../proxy/ssrf-protection).*

### Zero-Disk Configuration Storage
We have completely eliminated the local config file attack surface that almost every other secrets manager leaves open.
* **Zero plaintext keys on disk**: Sensitive authentication tokens and decrypted workspace keys are stored securely inside your OS Keychain (macOS Keychain, Windows Credential Manager, or Linux GNOME Keyring/KWallet via the `keychain-auth` daemon). `config.json` and `token.json` contain only empty values.
* **Automatic Migration**: Upgrading is completely seamless. Legacy config files are automatically read, migrated to the secure Keychain, and purged from disk files. You don't have to do anything but logout and login again.
* *Read more in [Keychain Storage Troubleshooting](../troubleshooting/keychain) and [Proxy Hardening](../guides/proxy-security-hardening).*

### Production Write-Protection
Actions affecting the production environment require local verification.
* **Local Password Verification**: Modifying production secrets or updating variables globally (`--all-envs`) prompts for the user password. The hash is verified locally using a PasswordHash derived from email and password (SHA-256).
* **Process Hijacking Defense**: Prevents rogue scripts, compromised packages, or misbehaving agents from modifying production variables.
* *Read more in [Environments Overview](../environments/overview).*

### Telemetry & Performance Insights
Monitor proxy usage and speed performance:
* **secrets_resolved**: Counts the number of individual secret values resolved by the proxy.
* **total_proxy_duration_ms**: Tracks request duration to provide speed insights.
* *Read more in [Proxy Performance](../proxy/performance).*

## Upgrading and Migration

### 1. Update the CLI
:::step
Download and install the latest agentsecrets binary (v3.0.0):
```bash
brew upgrade agentsecrets
# Or via npm/pip
npm install -g @the-17/agentsecrets
pip install --upgrade agentsecrets-cli
```
:::

### 2. Clear Stale Sessions and Establish Keychain Storage
:::step
Run logout to purge legacy disk-based credentials, then log in again. This automatically migrates your session to the secure native OS Keychain:
```bash
agentsecrets logout
agentsecrets login
```
You are now fully set up with the zero-disk, zero-knowledge v3.0.0 configuration storage.
:::
